Система обновлений SNAP CRM реализована по инкрементному принципу.
В этом случае компоненты обновляются по очереди от релиза к релизу и важно чтобы пользователь, пропустивший какие-либо обновления, получил их тоже.

Например, базовый инсталлятор содержит скомпилированные библиотеки Qt, которые изредка могут обновляться.
Предположим, что в релизе А эти библиотеки были обновлены, а в релизе Б обновлен только исполняемый файл программы. Если пользователь пропустит обновлени А, то наверняка программа перестанет запускаться.
Настроить загрузку файлов из обновления А можно с помощью добавления репозитория в файл Updates.xml обновления Б.
   <RepositoryUpdate>
    <Repository action="add" url="https://github.com/user/foobar/releases/download/release_x" displayname="qtlibupdate"/>
   </RepositoryUpdate>

   url также может быть на локальную файловую систему, например "file:///С:/foobar/release_x"
Эти параметры нужно добавлять в файл Updates.xml всех следующих релизов, т. к. любой пользователь может пропустить произвольное кол-во обновлений.
После очередного обновлении типа А в обновления типа Б нужно указать url на новый репозиторий.

Url, заданный в RepositoryUpdate, не записывается в файл maintanance.dat!
Для возможности использовать github.com в качестве сервера загрузок требуется модификация Qt Installer Framework (github не поддерживает директории в релизах). Модифицированную версию можно найти у меня в репозиториях.
